# OpenClaw Mission Control — Working Context

## Project Overview
Mission Control is a Python/FastAPI + Next.js 14 control plane for managing OpenClaw gateways, boards, agents, and tasks. It connects to OpenClaw gateways via WebSocket RPC.

## Repo Location
- **Root:** `/root/openclaw-mission-control/`
- **Sibling repo:** `/root/openclaw-zero-token/` (TypeScript OpenClaw fork)

## Tech Stack
- **Backend:** Python 3.12, FastAPI, SQLModel, Alembic, PostgreSQL 16, Redis 7
- **Frontend:** Next.js 16 (App Router), TypeScript, Tailwind CSS, React Query
- **Infrastructure:** Docker Compose (5 services: backend, frontend, db, redis, webhook-worker)

## Auth
- **Mode:** Local (token-based), NOT Clerk
- **Token location:** `.env` → `LOCAL_AUTH_TOKEN`
- **Frontend auth:** `src/auth/localAuth.ts` — stores token in browser sessionStorage
- **Auth header format:** `Bearer <token>`

## Key Architectural Patterns

### Gateway RPC
- All gateway communication goes through `backend/app/services/openclaw/gateway_rpc.py`
- `openclaw_call(method, params, config=config)` — the core RPC function
- Methods registered in `GATEWAY_METHODS` list
- WebSocket connection with device pairing or token auth

### Session Service
- `backend/app/services/openclaw/session_service.py` — `GatewaySessionService` class
- All gateway-related service methods live here
- Resolves gateway config via `resolve_gateway()` (board_id or direct URL)
